# Way SPN Media — органический спринт 01

Рабочий пакет для ежедневного продвижения Way SPN в Instagram Reels, TikTok и YouTube Shorts без рекламного бюджета.

## Цель спринта

- период: 16 июля — 14 августа 2026 года;
- темп: один исходный ролик в день, адаптированный под три площадки;
- аудитория: русскоязычные пользователи 18+;
- основной CTA: Telegram-бот `@WaySPN_robot`;
- главный KPI: оплаченные подписки;
- формат: 1080×1920, без ведущего, с крупными субтитрами и озвучкой.

`content-calendar.csv` содержит 30 утверждённых тем. `tracking-commands.txt` содержит команды создания трёх уникальных ссылок для каждого дня. `metrics.csv` — журнал результатов. Пакеты конкретных публикаций хранятся в `publishing/`.

## Источник истины для офферов

Перед рендером и публикацией цены сверяются с `config.py`, а скидки — с фактическим каталогом в боте. README проекта содержит устаревшие тарифы на 6 и 12 месяцев и для рекламных текстов не используется.

Актуальная базовая матрица на момент запуска:

| Вариант | 30 дней | 90 дней | Трафик | Устройства |
|---|---:|---:|---|---:|
| Обычная подписка | 200 ₽ | 500 ₽ | без заданного лимита по тарифу | до 5 |
| С антиглушилкой | 300 ₽ | 800 ₽ | 150 ГБ на каждый 30-дневный цикл | до 3 |

## Ежедневный процесс

1. Взять тему дня из `content-calendar.csv` и проверить цену/условия в рабочем каталоге.
2. Подготовить чистый MP4 без водяного знака другой площадки, обложку и три варианта текста.
3. Создать три tracking-ссылки командами из `tracking-commands.txt`.
4. Заменить шаблонные ссылки в публикационном пакете на ответы бота и проверить открытие каждой ссылки.
5. Загрузить ролик, заполнить описание и обложку, но не нажимать финальную кнопку без подтверждения владельца.
6. Заполнить `metrics.csv` через 24 часа и через 7 дней.

## Именование материалов

- видео: `dNN-slug-master.mp4`;
- обложка: `dNN-slug-cover.jpg`;
- платформенные версии: `dNN-slug-ig.mp4`, `dNN-slug-tt.mp4`, `dNN-slug-yt.mp4`;
- tracking-коды: `ig_dNN`, `tt_dNN`, `yt_dNN`.

## Метрики и решения

- `views`: просмотры площадки;
- `avg_watch_seconds`: среднее время просмотра;
- `completion_rate`: доля досмотров;
- `profile_actions`: переходы в профиль/на канал;
- `tracking_clicks`: переходы по уникальной ссылке;
- `paid_subscriptions`: оплаченные подписки;
- `subscription_revenue_rub`: выручка от подписок.

Главное сравнение — оплаченные подписки и выручка на 1 000 просмотров. Ролик с большим числом просмотров, но без переходов и оплат, не считается победителем.

Telegram-платёж получает код последнего активного tracking-перехода пользователя за 30 дней. Если такого перехода нет, используется исходный first-touch код пользователя. Поэтому новый клик существующего клиента можно связать с конкретным роликом.

## Безопасные формулировки

Использовать: «режим для сценариев с сетевыми ограничениями», «до 3/5 устройств», «150 ГБ на 30-дневный цикл», «после подтверждения оплаты ключ появляется в кабинете».

Не использовать без отдельного доказательства: «работает всегда», «100% обход», «не блокируется», «самый быстрый», «без потери скорости», «полная анонимность», «гарантированное подключение за 1–2 минуты», фальшивые отзывы и неподтверждённые счётчики клиентов.
